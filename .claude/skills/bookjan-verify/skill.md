# bookjan-verify — 품질 검증 가이드

bookjan42의 코드와 데이터 품질을 검증하는 체크리스트와 절차.

## 빌드 검증

```bash
npm run build    # Next.js 빌드 + 타입체크
```

빌드 실패 시:
1. 에러 로그의 파일:줄 확인
2. 타입 에러 → `types/index.ts` 또는 해당 컴포넌트 수정
3. import 에러 → 경로 확인 (`@/` 별칭 사용)
4. 3회 시도 후 실패 → 캐미에게 보고

## 코드 리뷰 체크리스트

### 보안
- [ ] 하드코딩 시크릿 없음 (NEXT_PUBLIC_* 환경변수 사용)
- [ ] Supabase anon key만 클라이언트에 노출 (서비스 키 금지)
- [ ] 사용자 입력이 SQL/HTML에 직접 삽입되지 않음
- [ ] AI 프롬프트에 외부 데이터 넣을 때 sanitize 적용

### 성능
- [ ] 서버 컴포넌트에서 할 수 있는 작업을 클라이언트로 옮기지 않음
- [ ] 불필요한 리렌더링 없음 (useMemo, useCallback 적절히)
- [ ] 카카오맵 SDK를 dynamic import로 로딩

### 스타일
- [ ] CSS 변수 사용 (하드코딩 색상 금지)
- [ ] 다크/라이트 모드 모두에서 텍스트 대비 AA 이상
- [ ] 라이트 모드 특히 주의 — 950/900 톤 사용하여 대비 확보
- [ ] 반응형: md 브레이크포인트 기준 맵/리스트 레이아웃 전환

### 데이터
- [ ] Supabase 쿼리에 에러 핸들링 있음
- [ ] `is_ad = false` 필터 적용 (리뷰 표시 시)
- [ ] FK 관계 정합성 (blog_reviews.place_id → places.id)

## UI 검증

### 다크 모드 체크
- 배경: #0C0A09 계열
- 텍스트: #FAFAF9 계열
- 카드: #1C1917 계열
- 액센트: #D97706 (amber)

### 라이트 모드 체크 (더 주의 필요)
- 배경: #FFFBF5 (cream)
- 텍스트: #1C1410 (dark brown) — 밝은 배경에서 대비 확인
- 배지/칩 색상이 배경과 충분히 구분되는지

### 반응형 체크
- 모바일 (<768px): 맵/리스트 토글
- 태블릿+ (≥768px): 맵+사이드바 나란히
- 필터 칩: 가로 스크롤, 스크롤바 숨김

## 크롤링 데이터 검증

새 크롤링 후 필수:
1. 최소 3~5건 원본 블로그 방문하여 교차 확인
2. content_summary가 실제 블로그 내용과 일치하는지
3. pros/cons가 블로그 내용에서 실제 추출 가능한지
4. is_ad 판별이 합리적인지
5. trust_score가 콘텐츠 품질을 반영하는지

## 배포 전 최종 체크

1. `npm run build` 성공
2. 로컬에서 `npm run dev` 후 주요 페이지 확인
3. 환경변수 설정 확인 (Vercel 대시보드)
4. 커밋 메시지 형식 준수
