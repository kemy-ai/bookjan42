## Phase 1: 기반 — 북바 지도 + 검색 (1~2주차)

### 목표
전국 북바를 지도에서 검색하고, 상세 정보를 확인할 수 있는 웹서비스

### 기능
- [ ] 북바 DB 구축 (캐미 수동 데이터 + 카카오맵 API 보조)
- [ ] 카카오맵 지도 연동 + 마커 표시 + 클러스터링
- [ ] 키워드 검색 (상호명, 지역)
- [ ] 필터 검색 (대화가능 여부, 분위기, 가격대, 지역)
- [ ] 장소 상세페이지 (메뉴, 가격, 분위기, 대화가능, 사진, 영업시간)
- [ ] 반응형 웹 디자인 (모바일 최적화)
- [ ] Vercel 배포

### 데이터
- Place (북바/서점)

### 인증
- 없음 (Phase 1은 비로그인으로 탐색만)

### "진짜 제품" 체크리스트
- [ ] Supabase에 실제 Place 데이터 저장 (목업 데이터 X)
- [ ] 카카오맵 API 실제 연동 (정적 이미지 X)
- [ ] Vercel에 배포되어 URL로 접속 가능
- [ ] 책익다 커뮤니티에 공유해서 피드백 수집

### Phase 1 시작 프롬프트
```
이 PRD를 읽고 Phase 1을 구현해주세요.
@PRD/01_PRD.md
@PRD/02_DATA_MODEL.md
@PRD/04_PROJECT_SPEC.md

Phase 1 범위:
- 북바 DB (Supabase) + 카카오맵 지도 + 검색/필터
- 장소 상세페이지
- 반응형 웹 + Vercel 배포

반드시 지켜야 할 것:
- 04_PROJECT_SPEC.md의 "절대 하지 마" 목록 준수
- 실제 Supabase DB 연결 (목업 X)
- 카카오맵 API 실제 연동
- Next.js 16 + shadcn/ui + Tailwind CSS
```

## Phase 2: AI 리뷰 수집 + 신뢰도 필터링 (3~4주차)

### 전제 조건
- Phase 1이 Vercel에 안정적으로 배포된 상태
- 최소 15곳 이상 Place 데이터 등록

### 목표
네이버 블로그 리뷰를 AI가 수집·분석하여 광고/진심을 분류하고, 장소페이지에 신뢰도 기반 리뷰 표시

### 기능
- [ ] 네이버 블로그 크롤링 파이프라인 (Scrapling + python3.12)
- [ ] AI 광고/진심 분류 모델 (AI Gateway)
- [ ] 블로거 신뢰도 점수 산정 알고리즘
- [ ] 장단점 자동 추출 (AI 요약)
- [ ] 장소 상세페이지에 AI 리뷰 섹션 추가
- [ ] 신뢰도 필터 (광고 숨기기 / 신뢰도순 정렬)

### 추가 데이터
- BlogReview (AI 수집 리뷰)

### 통합 테스트
- Phase 1 기능 (지도, 검색, 상세페이지)이 여전히 정상 동작하는지 확인

---

## Phase 3: 사용자 시스템 — 로그인 + 후기 + 문장 (5~6주차)

### 전제 조건
- Phase 1 + 2가 안정적으로 운영 중
- AI 리뷰 분석 품질 검증 완료

### 목표
사용자가 로그인하여 직접 솔직한 후기를 남기고, 책 속 문장을 수집하는 시스템

### 기능
- [ ] 소셜 로그인 (카카오 / 네이버 / 구글) — Supabase Auth
- [ ] 질문 기반 후기 작성 시스템
- [ ] 책 검색 + 후기에 책 연결 (Book_api 연동)
- [ ] 책 문장 남기기 (텍스트 직접 입력)
- [ ] 책 문장 사진 업로드 → OCR 텍스트 변환 (Tesseract.js)
- [ ] 포인트 적립 시스템
- [ ] 사용자 프로필 페이지

### 추가 데이터
- User, Review, Book, BookQuote

### 통합 테스트
- Phase 1 + 2 기능 정상 동작 확인
- 로그인 전/후 접근 권한 테스트

---

## Phase 4: 추천 알고리즘 + 커뮤니티 (7~8주차+)

### 전제 조건
- Phase 1 + 2 + 3 안정 운영
- 사용자 후기 최소 50건 이상 축적
- 회원 최소 100명 이상

### 목표
사용자 성향을 분석하여 맞춤 장소·책·메뉴를 추천하고, 비슷한 사람을 연결

### 기능
- [ ] 성향 프로필 설정 (UserTaste)
- [ ] "나는 이런 상황이야" → 맞춤 추천 (장소 + 책 + 메뉴)
- [ ] 비슷한 취향 사용자 매칭
- [ ] 모임/이벤트 기능
- [ ] 알림 시스템

### 추가 데이터
- UserTaste

### 주의사항
- AI 추천 비용 (AI Gateway 사용량에 따라 과금)
- 개인정보 처리 동의 필요 (성향 데이터 수집)
- 추천 알고리즘 품질은 데이터양에 의존 — 충분한 후기 축적 후 시작

---

## Phase 5: 수익화 + 앱 (추후)

### 전제 조건
- MAU 500명 이상
- 커뮤니티 활성화 확인

### 목표
지속가능한 비즈니스 모델 구축 + 모바일 앱

### 기능
- [ ] 수익 모델 확정 및 구현 (프리미엄/광고/커머스 중 선택)
- [ ] 결제 시스템 (Stripe)
- [ ] 모바일 앱 (React Native 또는 PWA)
- [ ] 독립서점 카테고리 확장
- [ ] 북바 파트너십 (제휴/예약)

### 주의사항
- 수익 모델은 Phase 1~3 피드백 기반으로 결정
- 앱 전환 시 PWA(웹앱을 앱처럼 설치)가 가장 비용 효율적
