## 1. 제품 개요

### 한 줄 요약
술과 독서를 사랑하는 사람들이 전국 북바를 발견하고, 솔직한 후기를 나누며, 자신에게 맞는 장소·책·사람을 추천받는 커뮤니티 플랫폼.

### 해결하는 문제
- 전국 북바(술+책방) 정보가 흩어져 있어 한눈에 찾기 어려움
- 네이버/블로그 리뷰가 광고성 후기로 가득해 진짜 정보를 판단하기 힘듦
- "나에게 맞는 북바"를 찾기 어려움 (대화 가능 여부, 분위기, 메뉴 등 개인 취향 반영 부족)
- 술과 독서를 좋아하는 사람들끼리 연결될 공간이 없음

### 핵심 가치
- **신뢰**: AI가 블로그 리뷰의 광고/진심을 분류하고, 블로거 신뢰도를 측정하여 가짜 후기를 걸러냄
- **개인화**: "나는 이런 상황이야"라고 말하면 장소+책+메뉴를 맞춤 추천
- **진정성**: 칭찬일색이 아닌, 장단점이 분명한 솔직한 후기 문화
- **연결**: 단순 장소 소개가 아닌, 몰입과 나다움을 중시하는 사람들의 커뮤니티

## 2. 사용자

### 주요 사용자
- **누구**: 20대 중반~30대 중반, 책과 술을 함께 즐기는 것에 관심 있는 사람
- **상황**: 혼자 조용히 책 읽을 북바를 찾거나, 친구와 대화 가능한 책방을 찾을 때
- **목표**: 나의 기분/상황에 맞는 북바를 발견하고, 다녀온 후 솔직한 경험을 남기고, 비슷한 취향의 사람과 연결됨

### 초기 사용자
- 책익다(Ikda) 커뮤니티 멤버 → 일반 대중 확장

### 사용자 시나리오

**시나리오 1: 탐색**
1. 사용자가 "서울 합정 근처 대화 가능한 북바"를 검색
2. 지도에 마커로 표시되고, 필터로 대화가능/분위기/가격대 조절
3. 관심 있는 곳의 상세페이지에서 AI가 분석한 솔직 리뷰와 메뉴/가격 확인

**시나리오 2: 후기**
1. 다녀온 후 질문 기반 후기 작성 ("혼자 갔나요?", "어떤 기분일 때 좋을까요?")
2. 마음에 든 책 문장을 사진으로 찍어 업로드 → OCR로 텍스트 변환
3. 솔직한 후기에 포인트 적립

**시나리오 3: 추천 (Phase 4)**
1. "오늘 혼자 조용히 책 읽고 싶어. 와인 한 잔 하면서"
2. 성향 프로필 기반으로 장소 + 해당 장소에 어울리는 책 + 메뉴 추천
3. 비슷한 취향의 다른 사용자 후기도 함께 표시

## 3. 핵심 기능

| 기능 | 설명 | 우선순위 | 복잡도 |
|------|------|----------|--------|
| 북바 지도 + 검색 | 전국 북바를 카카오맵에 표시, 키워드/위치/필터 검색 | P1 (Phase 1) | 보통 |
| 장소 상세페이지 | 메뉴, 가격, 분위기, 대화가능 여부, 사진, 영업시간 | P1 (Phase 1) | 간단 |
| AI 블로그 리뷰 수집 | 네이버 블로그 크롤링 → 광고/진심 분류 → 신뢰도 측정 | P1 (Phase 2) | 복잡 |
| 소셜 로그인 | 카카오/네이버/구글 계정으로 로그인 | P1 (Phase 3) | 보통 |
| 질문 기반 후기 작성 | 구조화된 질문으로 솔직한 장단점 유도 + 포인트 적립 | P1 (Phase 3) | 보통 |
| 책 문장 남기기 (OCR) | 책 속 문장 사진 → 텍스트 변환 → 장소+책 연결 | P2 (Phase 3) | 복잡 |
| 성향 기반 추천 | "나는 이런 상황이야" → 장소+책+메뉴 추천 | P2 (Phase 4) | 복잡 |
| 사용자 매칭 | 성향 유사한 사용자 연결, 모임 기능 | P3 (Phase 4) | 복잡 |
| 결제 시스템 | 프리미엄 기능 | P3 (Phase 5) | 복잡 |
| 모바일 앱 | 웹 → 앱 전환 | P3 (Phase 5) | 복잡 |

## 4. 사용자 흐름 (User Flow)

### 핵심 흐름: 북바 탐색
```
홈 → 지도/검색 → 필터 적용 → 마커 클릭 → 장소 상세 → AI 리뷰 확인 → 방문 결정
```

### 핵심 흐름: 후기 작성
```
장소 상세 → 후기 작성 → 질문 답변 → (선택) 책 문장 사진 업로드 → OCR 변환 → 제출 → 포인트 적립
```

### 상세 흐름
1. **홈**: 지도 중심 화면 + 인기 북바 + 최신 후기
2. **검색**: 키워드 입력 또는 필터(대화가능, 분위기, 가격대, 지역) 선택
3. **지도**: 카카오맵에 북바 마커 표시, 클러스터링
4. **장소 상세**: 기본 정보 + AI 분석 리뷰(장단점) + 사용자 후기 + 책 문장
5. **후기 작성**: 로그인 → 질문 응답 → 사진/문장 업로드 → 포인트 지급

## 5. 성공 기준

### Phase 1~2 (1개월, MVP)
- [ ] 전국 북바 30곳 이상 등록
- [ ] 지도에서 검색/필터가 정상 작동
- [ ] AI 블로그 리뷰가 최소 10곳에 표시
- [ ] 책익다 커뮤니티에서 10명 이상 접속

### Phase 3 (2개월차)
- [ ] 회원가입 50명 이상
- [ ] 사용자 후기 20건 이상
- [ ] 책 문장 10건 이상 수집

### Phase 4+ (3개월차~)
- [ ] 추천 알고리즘 만족도 설문 70% 이상 긍정
- [ ] 월간 활성 사용자(MAU) 500명 이상

## 6. 안 만드는 것 (Out of Scope)

> 이 목록에 있는 건 Phase 1~2에서 만들지 않습니다.

- 모바일 앱 -- 이유: 웹 우선, 반응형으로 충분
- 결제 시스템 -- 이유: 수익 모델 미정, MVP에서 검증 후 결정
- 사용자 매칭/모임 -- 이유: Phase 4에서 충분한 사용자 데이터 확보 후
- 독립서점 확장 -- 이유: 북바 먼저 검증 후 2단계로
- 실시간 채팅 -- 이유: 커뮤니티 기능은 후기+문장으로 시작
- 예약/주문 연동 -- 이유: 북바와 제휴 관계 필요, MVP 범위 초과
